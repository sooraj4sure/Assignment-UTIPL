#  Project Analytics Dashboard

A two-part data project combining **SQL Server analytics on the Northwind database** and a **live currency exchange rate tracker**, visualized through an interactive Streamlit dashboard.

---
| Streamlit Live Dashboard Demo | https://assignment-utipl.streamlit.app/ |

##  Installation

### 1. Clone or download this project



Or just download the ZIP and extract it into a folder.

### 2. Install Python dependencies

```bash
pip install streamlit pandas plotly pyodbc requests
```

### 3. Restore the Northwind database

1. Open **SSMS** and connect to `.\SQLEXPRESS`
2. Run this in a New Query window to create the database:

```sql
CREATE DATABASE Northwind;
GO
```

3. Go to **File → Open → File** and open `instnwnd.sql`  
   *(Download from: https://github.com/microsoft/sql-server-samples/tree/master/samples/databases/northwind-pubs)*
4. Change the database dropdown to `Northwind`
5. Press `Ctrl+A` then `F5` to execute
6. Verify with:

```sql
USE Northwind;
SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_TYPE = 'BASE TABLE';
```

You should see 13 tables including `Orders`, `Customers`, `Products`, `Categories`.

##  Running the Project

### Part 1 — SQL Queries (Northwind)

Open `northwind_queries.sql` in SSMS and run each section:

```sql
USE Northwind;

-- Query 1: Total revenue by product category
-- Query 2: Top 10 customers by lifetime order value
-- Query 3: All delayed orders (shipped > 7 days after order date)

-- Stored procedure (run once to create, then call anytime):
EXEC dbo.usp_NorthwindSalesReport;

-- With optional filters:
EXEC dbo.usp_NorthwindSalesReport
    @StartDate = '1997-01-01',
    @EndDate   = '1997-12-31',
    @DelayThreshold = 14,
    @TopN = 5;
```

### Part 2 — Currency Tracker

```bash
python currency_tracker.py
```

**Output files generated:**

| `currency_tracker.log` | Timestamped run history and any errors |

### Part 3 — Streamlit Dashboard

```bash

```

Opens automatically at `http://localhost:8501`

- Toggle between demo data and live SQL Server / live API in the sidebar

##  Scheduling (Windows Task Scheduler)

Run the setup script **once** to register a daily 8:00 AM job automatically:

```
Double-click  →  run_setup.bat
```

The script will:
- Auto-detect your Python installation
- Register `CurrencyTrackerDaily` in Windows Task Scheduler
- Offer an immediate test run
- Output all results to `scheduler_run.log`

**Manage the task later via PowerShell:**

```powershell
# Check status
Get-ScheduledTask -TaskName "CurrencyTrackerDaily"

# Run immediately
Start-ScheduledTask -TaskName "CurrencyTrackerDaily"

# Disable
Disable-ScheduledTask -TaskName "CurrencyTrackerDaily"

# Delete
Unregister-ScheduledTask -TaskName "CurrencyTrackerDaily"
```

##  Troubleshooting

**SSMS won't connect to SQL Server**
- Use server name `.\SQLEXPRESS`
- Set Encrypt → `Optional`
- Check `Trust Server Certificate`
- Make sure `SQL Server (SQLEXPRESS)` service is running in `services.msc`

**`pyodbc` connection error in dashboard**
- Ensure ODBC Driver 17 for SQL Server is installed
- Toggle off "Connect to live SQL Server" in sidebar to use demo data

**`currency_tracker.py` returns N/A**
- Check internet connection
- `frankfurter.app` may be temporarily down — try again in a few minutes
- The script logs errors to `currency_tracker.log`

**Task Scheduler runs but no CSV appears**
- Check `scheduler_run.log` for Python errors
- Make sure `requests` is installed: `pip install requests`
- Verify the path in `run_setup.bat` points to the correct folder

**Streamlit Cloud shows no SQL data**
- Expected — cloud can't reach your local SQL Server
- The SQL tab uses demo data automatically on cloud deployments
